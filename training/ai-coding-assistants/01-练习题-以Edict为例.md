# 练习题：以“三省六部（Edict）”为例

> 目标：让百官在真实项目里练手，不做空转。

## 练习 1（15 分钟）：加一个“版本信息”接口
- 目标：Dashboard 增加 `/api/version`，返回 git commit、启动时间。
- 验收：`curl http://127.0.0.1:7891/api/version` 返回 JSON。
- 要求：不影响现有接口；错误时返回合理提示。

## 练习 2（20 分钟）：把“旨意数量”展示改为可配置过滤
- 目标：页面显示“全部/仅JJC/含OC”三种视图切换。
- 验收：切换后数量与卡片列表正确。

## 练习 3（20 分钟）：修复一个可复现问题
- 目标：复现一次“重启后旨意卡片消失”，并加一条保护逻辑或告警。
- 验收：模拟重启后，卡片仍可见或至少提示如何恢复。

## 练习 4（30 分钟）：加一个“巡检摘要”卡片
- 目标：把最近一次巡检结论（从日志/状态）在 UI 顶部展示。
- 验收：每 30 分钟刷新一次摘要。

## 练习 5（30–45 分钟）：提交 PR 流程演练
- 目标：在 `jionpz/edict` 新建分支提交改动，写清楚 PR 描述。
- 验收：PR 中包含：目标、变更点、验收方法、风险与回滚。
