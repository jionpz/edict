# RFC：培训验收闭环（JJC-20260305-004）

- RFC 编号：JJC-20260306-005-RFC-001
- 发起人（中书省）：（代）太子
- 审议人（门下省）：（待门下补章）
- 落地人（尚书省）：（待指派）
- 日期：2026-03-06

## 1) 触发原因（必须）
- 触发现象：OpenCode/代码工具培训 **已有材料**，但一直缺“百官练习产出+回执+可验收证据包”，导致无法 Done、皇上问责。
- 触发指标：流程上属于“Done 无证据/在办无验收闭环”的典型。
- 严重级别：P1（影响组织执行力与可信度）。

## 2) 目标（1 句话，可验收）
在 24 小时内，让 `JJC-20260305-004` 达到“可验收完成”：**≥80% 百官提交回执**，且每条回执包含 PR/commit/diff + 验收命令输出，证据归档齐全。

## 3) 范围
### In scope
- 回执模板、截止时间、汇总目录、最小练习题、验收口径、统计表。

### Out of scope
- 不要求每官完成全部练习；不要求引入新平台/新工具（先把闭环做起来）。

## 4) 方案（不超过 1 页）
- 固定“最小练习”= 练习1（/api/version），允许其它练习替代，但必须可验收。
- 回执模板：使用 `data/tasks/JJC-20260305-004/03-打卡回执模板.md`。
- 证据归档：建立 `data/tasks/JJC-20260305-004/evidence/`，每官一份 `*.md` 回执。
- 汇总表：由礼部维护一张 `evidence/index.md`：官署/是否完成/链接/验收结果/备注。
- 截止与升级：
  - T0：今天 18:00 前提交回执；
  - T+2h：未交名单 @；
  - T+6h：仍未交升级到尚书省点名。

## 5) 验收（必须可复制）
- 验收步骤：抽检 3 份回执 + 全量统计。
- 验收命令（示例，至少具备其一）：
  - `curl http://127.0.0.1:7891/api/version` 返回 JSON
  - 或提供 PR/commit，并附 `pytest`/`npm test`/`pnpm test` 等项目自带验收命令
- 预期输出/截图要求：每份回执粘贴 ≤10 行关键输出。
- 证据归档路径：`data/tasks/JJC-20260305-004/evidence/`

## 6) 风险与回滚（必须）
- 风险：回执收集变成“形式主义”。
- 缓解：抽检必须跑命令；不合格回执退回重交。
- 回滚：无代码改动时无需回滚；若涉及仓库改动，统一用 git revert。

## 7) 实施计划
- Step 1（中书省）：下发统一口径（截止时间/模板/证据目录/最小练习）。
- Step 2（门下省）：审议并确认验收门槛（≥80% or 全员）。
- Step 3（尚书省）：督办执行与抽检验收，最终在看板标记 Done 并附证据路径。

## 8) 决议（门下省填写）
- 结论：待定
- 需补充材料：无
- 备注：建议从“先闭环”开始，避免一次性要求过高导致崩盘。
