# 制度/流程变更必须走三省六部（强制流程）

> 目的：制度优化也纳入可审计、可回滚、可验收的治理流程。

## 1) 何为“制度变更”
满足任一项即是制度变更：
- 群协作规则、路由规则、模板与话术
- 任务状态流转规则与职责边界
- 验收与证据标准（Done/Block）
- 指标阈值与告警分级
- 值守/巡检 cron 频率与范围
- 安全边界（allowFrom/groupPolicy/工具权限）

## 2) 强制流程（默认）
1) 中书省：发起 RFC（使用 `docs/governance/rfc_template.md`）
2) 门下省：审议结论（准/不准/需补充），明确风险与回滚
3) 尚书省：落地执行，提供验收证据
4) 太子：回奏皇上，并将变更写入 CHANGELOG

## 3) P0 快速修补例外
- P0（主链路断）允许尚书省先修复恢复服务
- 但必须在 24h 内补：RFC + 门下审议记录 + 验收证据

## 4) 验收要求
- 必须有：变更点清单、验收命令/步骤、关键输出/截图、回滚方案
- 无证据不得宣布完成
